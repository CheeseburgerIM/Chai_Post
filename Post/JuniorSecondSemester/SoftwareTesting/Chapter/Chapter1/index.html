Chapter01 - Introduction

对于软件故障我们能做什么？

  • 我们不能完全避免软件故障
  • 但是我们可以降低软件故障的风险
    • 更好的需求分析
    • 更好的软件设计
    • 更好的开发过程
    • 更好的项目管理
    • 更好的软件测试
    • …

为什么要做软件测试？

测试人员的目的是尽早消除软件中的错误

  • 提高软件质量
  • 减少修改代价
  • 维护用户满意度

这意味着什么？

软件测试变得越来越重要

什么是软件测试？

Verification + Validation(验证和确认)

  • 验证为通过软件说明书在开发团队内部判断软件产品是否正确
  • 确认是与用户进行沟通判断软件产品是否符合预期要求

基于测试成熟度的测试目标

  • LV0：软件测试等同于debugging
    • 软件测试应》debugging
  • LV1：测试的目的就是展示软件的正确性
    • 无法证明，只能说明软件产品在测试用例中的正确性
  • LV2：测试的目的就是找到软件中的故障
    • 具有一定负面影响，导致开发组与测试组对立
  • LV3：测试的目的并不是要证明具体的正确或失败，而是为了降低使用软件产品的风险
    • 公认软件中风险一定存在
  • LV4：测试是一种帮助所有IT领域工作者开发更高质量软件产品的一种思想
    • 拼写检查器被用于检测拼写错误，但其目的是为了提高人们拼写能力

为什么要学习软件测试？

  • 软件测试岗位需求大
  • 提高自身软件开发能力
  • 开发与测试无法分离

  • 更专业地开发软件
    • 专业人员
      • 按需完成
      • 按时完成
      • 保质完成
      • 风险可控
    • 业余人员
      • 靠英雄
      • 靠运气
      • 浪费
      • 不可控

  • 全方位的提升
    • 全栈技术
      • 编程、数据库、网络、操作系统、软件分析、计算、大数据、人工智能…
    • 批判性思维
    • 责任
    • 沟通技巧
    • 全方位思考
  • 质量思维
    • 不仅在软件领域
    • 生活、学习…
  • 软件测试的实践与科研紧密相关
    • 测试用例自动化生成
    • 自动化执行测试
    • 测试优化
      • 测试用例化简
      • 测试用例排序
      • 测试用例最小化
    • 错误定位
    • 自动化修复
    • 缺陷预测

从软件测试可以学到什么？

  • 测试覆盖
    • 图覆盖Graph
    • 逻辑覆盖Logic
    • Input Partition Coverage
    • Syntax Coverage
  • 测试设计与执行
    • 单元测试Unit
    • 接口测试Interface
    • 集成测试Integration
    • 压力测试Load
  • 测试驱动开发
  • 常见工具
    • JUnit
    • Selenium
    • JMeter
  • 前沿话题
    • 错误定位
    • 符号执行
    • 变异测试
    • 测试用例自动化生成
